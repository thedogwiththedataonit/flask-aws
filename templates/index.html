Alarms



Metric alarms

Relative threshold alarms that are set on metrics. Actions can be queued (Amazon SNS, EC2 Action, Autoscaling, incident report).

Composite alarms

Overarching alarm that is triggered by multiple other alarms. Meant to reduce alarm fatigue with common alarms by correlating them with other independent alarms.

Standard Metrics

Granularity from 1 to 5 minutes. CPU Utilization, Disk read/write, network in/out, and credit usage.

Custom Metrics

Custom metrics can be defined as either standard or high resolution. High-resolution metrics have a granularity of 1, 5, 10, 30, or 60 seconds. Calls PutMetricData API and are charged per metric.

Why

Datadog?


Hybrid Environments

Datadog is able to monitor any cloud environment and on-prem environment. CloudWatch struggles to monitor on-prem and lacks the flexibility for hybrid integrations.


One Dashboard, Multiple Vendors

As companies scale, it is common for their environments to stretch across multiple products and infrastructure from different cloud vendors. CloudWatch focuses on monitor AWS, Datadog is designed to monitor any cloud vendor along with over 400+ integrations.


DevOps Communciation

AWS requires permissions and AWS accounts to share dashboards, Alerting and sharing information on AWS is limited to AWS SNS and incident reports. Datadog has public links, social feed of alerts, incidents, and graphs that can be shared within the platform and across Pagerduty, Slack, ServiceNow.


Greater Granularity

Datadogs agent is able to pull metrics at 15 second granularity. CloudWatch's highest granularity for standard metrics are 1 minute.


Real User Monitoring

Cloudwatch does not have extnsive frontend support such as Datadog's RUM.


Superior UI

CloudWatch's dashboards and UI are nothing special.
