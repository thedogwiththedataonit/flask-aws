Offerings

Dashboard

CloudWatch

Instant AWS Integration

No aesthetic customizability, some what simple dashboard layout.

Alarms

Metrics and Composite

Metric alarms

Relative threshold alarms that are set on metrics. Actions can be queued (Amazon SNS, EC2 Action, Autoscaling, incident report).

Composite alarms

Overarching alarm that is triggered by multiple other alarms. Meant to reduce alarm fatigue with common alarms by correlating them with other independent alarms.

Logs

Cloudwatch

Query and Monitor Log Data

Error logs

Log retention is also tied to AWS storage services that can be from 1 day to 10 year retention.

Metrics

CloudWatch

Granularity

Base Resolution - 1 to 5 minutes
High Resolution - highest level of granularity, metric per second
Calls PutMetricData API and are charged per metric Metrics are default tracked characteristics of AWS services. CPU Utilization, Disk space/read/write, network in/out, instance health, and credit usage. These metrics can be grouped by service (EC2, RDS, S3), by autoscaling group, by instance type, by instance ID, etc.

Servicelens

CloudWatch

Application Traces

Gain visibility into application by correlating infrastructure monitoring (logs) and transaction monitoring (traces).

Pinpoint performance bottlenecks and reduce MTTR by speeding up root cause analysis.

Synthetics

CloudWatch

Canaries



Configurable scripts that run on a schedule to monitor endpoints and APIs. Much like Datadog's synthetics, they follow routes and perform actions on pages using puppeteer or selenium.

Why

Datadog?


Hybrid Environments

Datadog is able to monitor any cloud environment and on-prem environment. CloudWatch cannot monitor on-premise infrastructure as it was built for AWS, not the greater cloud.

One Dashboard, Multiple Vendors

As companies scale, it is common for their environments to stretch across multiple products and infrastructure from different cloud vendors. While CloudWatch can only monitor AWS, Datadog is able to monitor any cloud vendors infrastructure along with over 400+ integrations.

DevOps Communciation

AWS requires permissions and AWS accounts to share dashboards, Datadog offer public invite urls which allows anyone on any team to gain value from the platform. Alerting and sharing information on AWS is limited to AWS SNS and incident reports, however Datadogs social feed of alerts, incidents, and graphs can be shared within the platform and even across Pagerduty, Slack, ServiceNow.

Greater Granularity

Datadogs agent is able to pull metrics from applications at 15 second granularity. CloudWatch's highest granularity is at 1 minute for metrics and is charged per metric instead of Datadogs per host.

Superior UI

CloudWatch's dashboards and UI are nothing special.
